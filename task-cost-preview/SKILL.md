# 任务成本预估

在执行任务前，评估并展示本次任务的预期成本，让用户决定是否继续。

## 第一步：分析任务复杂度

根据任务描述，判断复杂度等级：

| 等级 | 特征 | 典型输出 token |
|------|------|---------------|
| 简单 | 单步回答、短解释、单文件小改 | 200–800 |
| 中等 | 写代码、分析文档、多步工具调用 | 800–3,000 |
| 复杂 | 多文件操作、完整功能开发、深度研究 | 3,000–10,000 |
| 极复杂 | 完整项目构建、大规模重构、长链调用 | 10,000–30,000 |

## 第二步：估算 token 数

**输入 token（凭对话上下文感知估算）：**
- 当前对话上下文：直接凭感知给出粗略估算（中文约 1.5字/token，英文约 4字/token）
- 任务本身的描述 token 已包含在上下文中

**输出 token：**
按复杂度等级取中位数估算，考虑任务的具体情况适当调整。

## 第三步：计算费用与耗时

**定价（claude-sonnet-4-6）：**
- 输入：3.00 USD / 1M tokens
- 输出：15.00 USD / 1M tokens

**输出速度约 80 tokens/秒**，用于估算耗时。

计算公式：
- 输入成本 = 输入tokens × 3 ÷ 1,000,000 USD
- 输出成本 = 输出tokens × 15 ÷ 1,000,000 USD
- 预计耗时 = 输出tokens ÷ 80（秒）

## 注意事项

- 估算完成后，**必须等待用户明确确认**（如"继续"、"好的"、"yes"）再开始执行任务。
- 如果任务有明显分支（例如"可能需要也可能不需要大量网络搜索"），给出两个场景的估算范围。
- 如果任务描述非常模糊，先用一句话确认理解后再估算，不要展开执行。
- 不使用 emoji，保持简洁的表格格式。
- 全程使用中文输出（除 token/$ 等技术术语外）。
